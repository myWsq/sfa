# SFA v0.2.0 Release Notes Draft

## Highlights

- Close out M1 as a stable minimal usable chain with executable repository-level verification gates.
- Keep the frozen `format-v1` protocol, canonical golden corpus, and CLI regression coverage aligned with the shipped repository state.
- Publish the current release checklist and milestone status as repository-traceable artifacts instead of leaving them implicit in scattered docs.

## Compatibility

- No wire-format change is introduced in this release-prep slice; `format-v1` remains the frozen compatibility boundary for SFA v1.
- Existing `sfa pack` / `sfa unpack` workflows, supported codecs, and current Unix entry semantics remain unchanged.
- This release candidate does not expand v1 scope to xattrs, ACLs, or non-Unix behavior parity.

## Verification

The authoritative release checklist was executed successfully on 2026-03-17:

- `cargo fmt --all --check`
- `cargo test --workspace`
- `bash tests/scripts/run_protocol_smoke.sh`
- `bash tests/scripts/run_streaming_smoke.sh`
- `bash tests/scripts/run_safety_smoke.sh`
- `bash tests/scripts/run_roundtrip_smoke.sh`
- `cargo run -p sfa-bench --bin tar_vs_sfa -- --dry-run --output benches/results/latest.json`

Benchmark dry-run remained mandatory for release readiness. A fresh committed benchmark baseline refresh was not required for this change because the release-prep work did not alter benchmark logic, committed datasets, codec integration, planner behavior, or benchmark result schema.

## Known Gaps

- M3 Unix semantics work remains deferred, including stronger metadata restore validation and any decision on xattrs / ACL follow-up.
- The repository still uses a manual Git tag + GitHub Release flow; automated binary packaging and crates.io distribution are still out of scope.
- Large real-world thread sweeps remain diagnostic evidence rather than a formal release gate.
