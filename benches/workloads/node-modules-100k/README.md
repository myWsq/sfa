# node-modules-100k

Purpose: provide the default benchmark workload for SFA by generating a deterministic `node_modules`-style deep-directory tree that stresses many-small-files pack and unpack behavior under default command parameters.

Construction:

- generated offline from the committed `recipe.json` contract and text templates under `templates/`
- does not download packages or depend on a package manager registry
- models `npm` / `yarn classic` style nested dependency subtrees instead of flat repeated directory copies

Workload contract:

- `10,560` generated package directories under nested `node_modules/` subtrees
- `105,601` regular files including the root manifest
- package nesting depth of `5`
- dominant file types: `.js`, `.json`, `.d.ts`, `README.md`, `LICENSE`

Interpretation notes:

- this workload is intended as the benchmark headline path, not as a checked-in fixture tree
- actual generated byte size is recorded by the benchmark report and committed baseline asset
- benchmark diagnostics should treat repeated runs as cache-sensitive and compare thread sweeps with consistent cache-warming policy
